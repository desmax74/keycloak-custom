Authentication

Dell Software Developers Platform use OAuth 2.0 for authorization. Follow the steps below to get authorized.

1. Get an API Account

Register for your Dell Software Developers account.

2. Request Access Token

Access token is unique to a user and their role. Some services requires OAuth2 authentication, in these cases, the headers must include a valid authentication token.

Example HTTP headers: Accept: application/json Authorization: Bearer {access token}

3. Make the call

Once you have successfully obtained an access token, you can start making authenticated API requests.

See examples for more information.

Response status codes

Success

Standard HTTP response codes are returned by the server indicating the outcome of the request. Please consult each REST service for their specific return code.

Here are the status codes most commonly used by our REST services:

200 Request succeeded 202 Your request to a POST or PUT was successful. 403 Access denied. This means you do not have the right permission to invoke the service. 404 The resource was not found, this could be because there are errors in the request parameters. 405 Usually this means you used the wrong HTTP method (e.g. GET instead of POST).

GET /locations?ipaddress Get Geolocation Service

Get location service. This call does not require any authentication.

Request

  • ipaddress The IP address (e.g. 13.144.12.5).

Response

Return the geo information of the specified IP.

Status: 200 OK 204 IP Address not found

{ "postalCode": "06850", "country": "United States", "city": "Norwalk", "countryCode": "US", "latitude": 41.1271, "longitude": -73.4416, "paymentCurrency": "USD" }

POST /sms?mobile={number}&message={message} Send SMS message

Send a SMS message.

Authentication

You must use a valid access token to make this authenticated call.

Request

  • mobile Mobile number to send the text message (e.g. +14158141829).
  • message The text message.

Response

If succeeds, the text message will be delivered to the mobile phone.

Status: 202 OK 400 Invalid phone number

GET /cloudproviders Get Cloud Providers

Query all the cloud providers supported by the Dell Cloud Marketplace. This call does not require any authentication.

Response

[ { "name": "AWS", "id": 1 }, { "name": "Google", "id": 2 }, { "name": "Azure", "id": 3 }, { "name": "Joyent", "id": 4 } ]

GET /cloudproviders/{id} Get Cloud Providers by Id

Get a cloud provider supported by the Dell Cloud Marketplace by its Id. This call does not require any authentication.

Request

  • id The Cloud Provider Id.

Response

{ "name": "AWS", "id": 1 }

POST /isvblueprint Add ISV Blueprint

Enables authorized partners to post their blueprints to Dell Cloud Marketplace. You must contact Dell Software and sign an agreement before you can post your blueprints.

Request

You must use a valid access token with extended rights to make this authenticated call.

Parameters:

{ "name": "", "id": 0 }

Response

If succeeds, the blueprint will be added to Dell Cloud Marketplace.

Status: 200 Blueprint created successfully. 403 Only authorized Dell partners can add a new blueprint.

PUT /isvblueprint/{id} Update ISV Blueprint

Enables authorized partners to update their blueprint from Dell Cloud Marketplace

Request

You must use a valid access token with extended rights to make this authenticated call.

Response

If succeeds, the blueprint will be updated.

Status: 200 Blueprint updated successfully. 204 Blueprint does not exist. 403 Only authorized Dell partners can update there existing blueprint.

DELETE /isvblueprint/{id} Delete ISV Blueprint

Enables authorized partners to delete their blueprint from Dell Cloud Marketplace.

Request

You must use a valid access token with extended rights to make this authenticated call.

Response

If succeeds, the blueprint will be deleted.

Status: 200 Blueprint deleted successfully. 204 Blueprint does not exist. 403 Only authorized Dell partners can delete there existing blueprint.
