Authentication

Dell Software Developers Platform use OAuth 2.0 for authorization. Follow the steps below to get authenticated.

1. Get an API Account

2. Request Access Token

3. Make the call

Response status codes

Success

Successes differ from errors in that their body may not be a simple response object with a code and a message. The headers however are consistent across all calls:

  • GET, PUT, DELETE returns 200 OK on success,
  • POST returns 201 on success,

Status: 200 OK

Error

Error responses are simply returning standard HTTP error codes along with some additional information:

  • The error code is sent back as a status header,
  • The body includes an object describing both the code and message (for debugging and/or display purposes),

For a call with an invalid authentication token for example:

Status: 403 Access denied { status: 403, message: 'Access denied: you do not have permission to access the service' }

POST /isvblueprint Add ISV Blueprint

Enables authorized partners to post their blueprint to Dell Cloud Marketplace

Request

  • The headers must include a valid authentication token.
  • The body can’t be empty and must include at least the name attribute, a string that will be used as the name of the thing.

Authentication: bearer TOKEN { name: 'My new thing' }

Response

If succeeds, returns the created thing.

Status: 201 Created { id: new_thing, name: 'My new thing' }

PUT /isvblueprint/:id Update ISV Blueprint

Enables authorized partners to update their blueprint from Dell Cloud Marketplace

Request

  • :id is the id of the blueprint to update.
  • The headers must include a valid authentication token.
  • The body can’t be empty and must include at least the name attribute, a string that will be used as the name of the thing.

Authentication: bearer f862f658-ad89-4fcb-995b-7a4c50554ff6 { name: 'My new thing' }

Response

Sends back a collection of things.

Status: 200 OK { { id: thing_1, name: 'My first thing' }, { id: thing_2, name: 'My second thing' } }

DELETE /isvblueprint/:id Delete ISV Blueprint

Enables authorized partners to delete their blueprint from Dell Cloud Marketplace.

Request

  • :id is the id of the blueprint to delete.
  • The headers must include a valid authentication token.
  • The body is omitted.

Response

Sends back a collection of things.

Status: 200 Deleted { code: 200, message: 'Your blueprint (id:123) was deleted' }

GET /applications/:name Get Application Metrics

Get application metrics

Request

  • :name is the name of the application
  • The headers must include a valid authentication token.

Authentication: bearer f862f658-ad89-4fcb-995b-7a4c50554ff6 { name: 'My new thing' }

Response

Sends back the application metrics

Status: 200 OK { { id: thing_1, name: 'My first thing' }, { id: thing_2, name: 'My second thing' } }
